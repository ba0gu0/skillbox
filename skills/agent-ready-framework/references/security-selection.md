# 安全文档选择

安全要求必须与真实攻击面匹配。
通用红线可以放在 Agent 入口或 DoD；只有项目特有、需要反复执行的
高风险检查才单独创建 security checklist。

## 选择信号

| 信号 | 需要关注的边界 |
|------|----------------|
| 登录、会话、角色、管理后台 | 认证、授权、会话、审计 |
| PII、财务、医疗、密钥 | 分类、最小权限、加密、保留和脱敏 |
| 多租户 | 租户隔离、跨租户查询、配额和审计 |
| 支付或资产变更 | 服务端金额、签名、幂等、二次确认 |
| 文件上传/下载 | 大小、类型、内容探测、路径和存储 |
| 外部 URL、代理、抓取 | SSRF、重定向、DNS、凭证和响应头 |
| shell、插件、代码执行 | 命令注入、沙箱、资源限制和授权 |
| webhook/OAuth | 签名、重放、state、redirect URI 和 token |
| 公网 API | 鉴权、速率限制、输入、错误和版本兼容 |
| 供应链或发布 | 依赖来源、签名、权限、secret 和构建产物 |

没有对应信号时删除 SQL 注入、XSS、认证、数据库、HTTPS 等不适用检查项。

## 三类规则

- **禁止**：违反高优先级指令、泄露 secret、绕过授权、破坏数据完整性等
  不可接受行为。
- **需确认**：不可逆删除、生产变更、权限扩大、真实外部副作用或
  冲突意图选择。
- **必须**：与改动区域匹配的验证、secret 脱敏、必要文档写回和
  剩余风险披露。

不要把添加普通依赖、修改普通文件或执行用户已明确要求的范围内工作
一律列为需确认。
是否询问取决于可逆性、外部副作用和授权范围，
而不是模板中的固定清单。

## 文档分工

- Agent 入口：不可违反的项目红线和需要额外授权的操作。
- DoD：完成某类变更前需要运行的验证。
- security checklist：项目特有、可重复、需要逐项复核的高风险控制。
- threat model/安全审计报告：威胁、攻击路径、发现和修复建议；
  不要塞进日常清单。

本技能只负责选择和路由安全文档，不声称完成专业安全审计。
用户要求威胁建模、OWASP 审计或合规审计时，
应使用对应安全技能并把结论链接回项目文档。
