# 任务状态语义

只有项目需要跨会话或多人/多 Agent 跟踪任务时才使用统一状态机。

| 状态 | 含义 | 进入条件 |
|------|------|----------|
| Draft | 尚在澄清，不能直接领取 | 范围、依赖或验收仍不完整 |
| Ready | 可以执行 | 范围和验收明确，必要依赖已满足 |
| In Progress | 正在执行 | 已有明确执行者开始工作 |
| Blocked | 暂时无法继续 | 阻塞原因、解除条件和责任方已记录 |
| Done | 已完成 | 实现、相关验证和必要写回均完成 |

## 流转规则

- Draft 只能在范围、依赖和验收明确后进入 Ready。
- Ready 领取后进入 In Progress；不要用 In Progress 表示“有人可能会做”。
- 非 Done 状态都可进入 Blocked；解除后回到阻塞前状态，
  无法判断时回到 Ready。
- 验证失败时保持 In Progress 或进入 Blocked，不得标记 Done。
- Done 不是提交存在或代码写完；
  必须有当前树、验证结果和必要文档写回支持。

## 权威来源

- 启用独立任务卡时，任务卡是单任务状态的权威来源；
  队列只提供索引和排序。
- 未启用任务卡时，next actions 或 current state 是权威来源。
- 不要在 current state、next actions、spec task 和 handoff 中
  维护互相独立的状态副本。
- 视图需要展示状态时应链接到权威来源，
  或在同一次改动中同步并验证一致性。

项目生命周期阶段与任务状态不同。
生命周期描述产品整体所处阶段，应从项目事实推导，
不要为了套用状态机强制选择固定阶段名称。
