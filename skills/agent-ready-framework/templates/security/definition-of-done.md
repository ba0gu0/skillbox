# 完成定义

> 每个任务完成必须满足以下标准

---

## 核心原则

**未验证不得 Done**：所有任务必须经过验证才能标记为完成。

**完成后必须写回状态**：任务完成后必须更新相关状态文档。

---

## 通用标准

每个任务完成必须满足：

- [ ] 代码编译/构建通过
- [ ] 相关测试通过
- [ ] 无 lint 错误
- [ ] 无安全漏洞
- [ ] 文档已更新（如涉及）
- [ ] 安全检查清单通过

---

## 按任务类型

- 无数据库时删除 SQL 注入。
- 无 HTML/浏览器输出时删除 XSS。
- 无认证/权限模型时删除权限检查。

### 后端任务

```bash
# 必须运行（根据项目技术栈填写真实命令）
{TEST_CMD}
{LINT_CMD}
```

检查项：
- [ ] 错误处理完整
- [ ] 日志记录合理
- [ ] 无硬编码配置
- [ ] API 响应格式正确

### 前端任务

```bash
# 必须运行（根据项目技术栈填写真实命令）
{TEST_CMD}
{LINT_CMD}
{BUILD_CMD}
```

检查项：
- [ ] 响应式适配
- [ ] 无 console 错误
- [ ] 可访问性符合标准
- [ ] 组件复用合理

### CLI/脚本任务

- [ ] 参数验证完整
- [ ] 文件路径处理无路径遍历风险
- [ ] 错误输出清晰，退出码符合约定
- [ ] 临时文件已清理
- [ ] 输出不包含敏感信息

### 数据库任务

- [ ] 迁移脚本可回滚
- [ ] 索引合理
- [ ] 无 N+1 查询
- [ ] 数据完整性约束

### API 任务

- [ ] 接口文档已更新
- [ ] 错误码定义完整
- [ ] 请求/响应格式正确
- [ ] 权限检查正确

### 配置任务

- [ ] `.env.example` 已同步
- [ ] 配置项有默认值
- [ ] 配置变更已文档化

### 文档任务

- [ ] 内容准确完整
- [ ] 格式规范一致
- [ ] 索引已更新

---

## 安全标准

所有任务必须满足：

- [ ] 无敏感信息硬编码
- [ ] 无 SQL 注入风险
- [ ] 无 XSS 风险
- [ ] 无命令注入风险
- [ ] 输入验证完整
- [ ] 权限检查正确
- [ ] 日志不包含敏感信息

---

## 文档标准

涉及以下变更时需更新文档：

| 变更类型 | 需更新文档 |
|---------|-----------|
| 新功能 | `{SPECS_PATH}`, `{TASKS_PATH}`, `{CURRENT_STATE_PATH}` |
| 架构决策 | `{ADR_PATH}` |
| 配置变更 | `.env.example`, docs/ |
| API 变更 | API 文档 |
| 行为变更 | AGENTS.md (如涉及红线) |

---

## 完成后必须

1. 运行所有验证命令
2. 完成 `{SECURITY_PATH}` 检查
3. 更新 `{CURRENT_STATE_PATH}`
4. 如项目存在独立任务队列，更新 `{NEXT_TASKS_PATH}`
5. 如项目存在任务卡，更新对应 `{TASKS_PATH}` 文件
6. 汇报变更文件列表
7. 说明剩余风险（如有）

---

## 验证失败处理

如果验证命令失败：

1. **不要标记任务为 Done**
2. 记录失败原因
3. 修复问题
4. 重新运行验证
5. 如无法修复，标记为 Blocked 并说明原因
