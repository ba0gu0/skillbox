## 使用说明

1. 任务完成后，逐项检查以下清单
2. 如有任何一项不通过，必须修复后才能标记任务完成
3. 如有新增安全风险，添加到本清单
4. 完成后在任务卡或任务队列中标记安全检查通过

## 代码安全

### 输入验证

- [ ] 所有用户输入已验证
- [ ] 输入长度已限制
- [ ] 特殊字符已处理
- [ ] 文件上传类型已限制

### 注入防护

- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 无命令注入风险
- [ ] 无 XSS 风险（输出已转义）
- [ ] 无路径遍历风险

### 认证授权

- [ ] 敏感操作有权限检查
- [ ] 会话管理正确
- [ ] 无越权风险
- [ ] Token 有过期时间

### 错误处理

- [ ] 错误信息不泄露敏感信息
- [ ] 异常已正确处理
- [ ] 日志不包含敏感信息
- [ ] 错误有合理用户提示

---

## 凭证安全

### 存储

- [ ] 无硬编码凭证
- [ ] 敏感配置使用环境变量
- [ ] `.env` 文件在 `.gitignore` 中
- [ ] 密钥使用安全存储

### 传输

- [ ] 敏感数据使用 HTTPS
- [ ] 无明文传输密码
- [ ] API Key 不在 URL 中
- [ ] 敏感请求有签名

### 访问

- [ ] 最小权限原则
- [ ] 敏感文件有访问控制
- [ ] 审计日志完整
- [ ] 凭证有定期轮换机制

---

## 依赖安全

- [ ] 新增依赖已验证来源
- [ ] 无已知漏洞依赖
- [ ] 依赖版本锁定
- [ ] 无不必要的依赖

---

## 数据安全

### 存储

- [ ] 敏感数据已加密
- [ ] 数据库访问有权限控制
- [ ] 备份已加密

### 处理

- [ ] 不记录敏感数据
- [ ] 不在日志中输出敏感信息
- [ ] 临时文件已清理

## 安全问题处理

4. 制定修复方案
5. 修复并验证
6. 更新本检查清单
7. 通知相关人员
