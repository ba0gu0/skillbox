## 写入规则

### 何时写入

遇到以下情况时，必须记录到本文件：

1. **编译/构建失败** - 花费 > 10 分钟排查
2. **测试失败** - 根因不明显
3. **安全相关** - 任何凭证、权限、注入问题
4. **线上问题** - 影响用户的问题
5. **重复犯错** - 同类问题出现 2 次以上

### 何时提升到 AGENTS.md

当错误满足以下条件时，应提升为 AGENTS.md 规则：

1. **通用性** - 其他项目也可能遇到
2. **严重性** - 可能导致安全或数据问题
3. **频率** - 同类问题出现 2 次以上

提升格式：
```markdown
在 AGENTS.md 红线部分添加：
- <简短规则描述>
```

### 记录格式

```markdown
### {DATE} - {ERROR_TITLE}

**场景**: <在什么情况下发生>
**现象**: <错误表现>
**根因**: <根本原因>
**解决**: <如何解决>
**教训**: <如何避免>
**规则**: <转化为 AGENTS.md 的规则，或写"无需提升">
```
