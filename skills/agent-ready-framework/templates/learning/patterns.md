## 写入规则

### 何时写入

遇到以下情况时，应记录到本文件：

1. **复杂问题解决** - 花费 > 30 分钟设计/实现
2. **性能优化** - 有可衡量的改进
3. **架构决策** - 涉及多个模块的设计
4. **复用价值** - 其他项目/模块可能用到

### 何时提升到 AGENTS.md

当模式满足以下条件时，应提升为 AGENTS.md 规则：

1. **通用性** - 适用于项目中所有类似场景
2. **重要性** - 影响代码质量或开发效率
3. **频率** - 在项目中多次应用

提升格式：
```markdown
在 AGENTS.md 代码风格部分添加：
- <简短模式描述>
```

### 记录格式

````markdown
### {PATTERN_NAME}

**场景**: <适用于什么场景>
**方案**: <解决方案>
**代码**:
```
<代码示例>
```
**效果**: <实际效果>
**适用条件**: <什么情况下使用>
**提升**: <是否需要提升到 AGENTS.md>
````
