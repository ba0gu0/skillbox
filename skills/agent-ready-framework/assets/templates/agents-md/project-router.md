1. 开始前检查工作区状态和适用的局部指令。
2. 从当前状态和下一步行动的权威文件确定范围与依赖。
3. 保留用户改动，只修改任务所需文件。
4. 按改动区域运行验证；没有运行的检查必须明确披露。
5. 行为、架构、配置、任务状态或长期规则变化时更新对应权威文档。
