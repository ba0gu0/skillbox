# 完成定义

## Done 的含义

任务只有在范围内实现、相关验证完成、必要文档或状态写回完成后，
才能标为 Done。
未运行的检查必须写明“未运行”及原因，不能等同于通过。

## 证据与写回

完成时至少记录：

- 实际修改的文件与范围。
- 已运行验证的命令、退出结果和必要摘要。
- 未运行验证及原因。
- 行为、配置、架构、任务状态或长期规则变化对应的权威文档更新。
- 剩余风险和后续行动。

## 失败处理

验证失败时保持 In Progress，或在存在外部阻塞时标为 Blocked。
不要删除失败测试、降低验收标准或伪造结果来满足 Done。
