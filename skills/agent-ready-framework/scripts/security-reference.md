# 项目类型安全红线

> 根据项目类型，生成对应的安全检查项

---

## 通用安全红线（所有项目）

### 禁止操作

- 不提交 `.env` 文件或 `secrets/` 目录
- 不硬编码敏感信息（密码、Token、API Key）
- 不在日志中输出敏感信息
- 不跳过安全检查标记任务完成

### 必须操作

- 运行测试后才能提交
- 完成任务后更新状态文档
- 安全检查通过后才能标记 Done

---

## Small 项目安全红线

适用：CLI 工具、脚本、POC、学习项目

### 禁止操作

- 不读取用户敏感文件（如 `.ssh/`、`.gnupg/`）
- 不在临时文件中存储敏感数据
- 不绕过用户确认执行危险操作

### 必须检查

- 命令行参数验证
- 文件路径验证（无路径遍历）
- 输出不含敏感信息

---

## Medium 项目安全红线

适用：Web 应用、API 服务、管理后台。移动后端按 API 服务处理，并补充速率限制、
客户端版本兼容、推送通知凭证等检查。

### 禁止操作

- 不在前端存储 Token 或敏感数据
- 不在 API 响应中返回敏感字段
- 不信任前端传入的用户身份
- 不使用明文存储密码

### 必须检查

**认证安全**：
- [ ] 登录态检查正确
- [ ] Token 有过期时间
- [ ] 如使用 Session，有合理超时；纯 API 服务检查 Token 刷新和吊销策略

**API 安全**：
- [ ] 所有 API 有权限验证
- [ ] 无越权访问风险
- [ ] 请求参数已验证
- [ ] 响应不含敏感字段
- [ ] 关键接口有速率限制或滥用防护

**数据安全**：
- [ ] 密码使用 bcrypt/argon2
- [ ] 敏感数据加密存储
- [ ] 数据库查询无注入风险

---

## Large 项目安全红线

适用：微服务、多租户平台、企业系统、金融系统

### 禁止操作

- 不在无 ADR 的情况下修改安全相关代码
- 不绕过 DoD 验收流程
- 不跨租户访问数据
- 不在日志中记录 PII（个人身份信息）

### 必须检查

**多租户安全**：
- [ ] 租户隔离正确
- [ ] 无跨租户数据泄露
- [ ] 租户资源配额限制

**微服务安全**：
- [ ] 服务间通信加密
- [ ] 服务认证正确
- [ ] 无内部 API 暴露到外部

**审计合规**：
- [ ] 操作日志完整
- [ ] 审计记录不可篡改
- [ ] 关键操作有二次确认

**数据安全**：
- [ ] 数据分类正确（敏感/非敏感）
- [ ] 数据脱敏规则完整
- [ ] 数据导出有审批流程

## 特殊场景安全红线

### 代理/网关服务

**核心红线**：
- 官方凭证只在服务端，不暴露给浏览器
- 不透传上游 `Set-Cookie`
- 不在日志中记录 Cookie/Token
- 会话隔离，无串线风险

**必须检查**：
- [ ] 反代响应已清理敏感头
- [ ] SSE 透传不泄露凭证
- [ ] WebSocket 代理已验证
- [ ] 文件上传不落盘

### 文件上传服务

**核心红线**：
- 文件类型验证在服务端
- 文件大小有限制
- 不执行上传的文件
- 不存储敏感文件名

**必须检查**：
- [ ] 文件类型白名单
- [ ] 文件大小上限
- [ ] 文件名已清理
- [ ] 存储路径无路径遍历

### OAuth 认证服务

**核心红线**：
- Client Secret 只在服务端
- State 参数已验证
- Redirect URI 白名单
- Token 不在前端存储

**必须检查**：
- [ ] OAuth State 验证
- [ ] Redirect URI 白名单
- [ ] Token 存储安全
- [ ] 刷新 Token 流程正确

### 支付系统

**核心红线**：
- 支付凭证加密存储
- 金额计算在服务端
- 支付状态有二次验证
- 不信任前端传入的金额

**必须检查**：
- [ ] 支付凭证加密
- [ ] 金额计算服务端
- [ ] 支付回调验证签名
- [ ] 交易日志完整
